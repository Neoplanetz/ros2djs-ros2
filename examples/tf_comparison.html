Setup (ROS 2, 2 terminals besides rosbridge)

  1. Static transform — place test_frame at (3, 2) in map: 
    ros2 run tf2_ros static_transform_publisher --x 3 --y 2 --z 0 --roll 0 --pitch 0 --yaw 0 --frame-id map --child-frame-id test_frame
  2. Publish a marker at the origin of test_frame (1 Hz): 
    ros2 topic pub /tf_demo_markers visualization_msgs/msg/MarkerArray "{markers: [{header: {frame_id: 'test_frame'}, ns: 'demo', id: 1, type: 1, action: 0, pose: {position: {x: 0.0, y: 0.0, z: 0.0}, orientation: {w: 1.0}}, scale: {x: 0.5, y: 0.5, z: 0.5}, color: {r: 1.0, g: 0.2, b: 0.2, a: 1.0}}]}" -r 1

Expected: red square on the left at (0, 0) (frame ignored). Same square on the right at (3, 2) (TF applied).

If the right canvas is empty: check the status bar. tf frames: (none yet) means TF not reaching us; rendered=(...) with visible=true means it's there but off-screen — use the pan/zoom to find it, or click "Inject fake" to confirm the code path works without ROS.

Multi-robot scenario

For the multi-robot case, run two static publishers and one MarkerArray whose markers use mixed robot_0/odom and robot_1/odom frame_ids. Without tfClient they collapse to one spot; with tfClient they separate.

ros2 run tf2_ros static_transform_publisher --x  5 --y 0 --z 0 --roll 0 --pitch 0 --yaw 0 --frame-id map --child-frame-id robot_0/odom
ros2 run tf2_ros static_transform_publisher --x -5 --y 0 --z 0 --roll 0 --pitch 0 --yaw 0 --frame-id map --child-frame-id robot_1/odom
